#DesafioCreativedrive

O PRESENTE SISTEMA É UMA IMPLEMENTAÇÃO DE MICROSERVIÇO
FEITO EM SPRINGBOOT E CONSUMINDO UM BANCO DE DADOS MONGOBD

FOI REALIZADO PARA ATENDER UM DESAFIO PARA A CREATIVEDRIVE

O SISTEMA FORNECE UM MICROSERVIÇO COM ENDPOINTS DE AUTENTICAÇÃO
E GERENCIA (CRUD) DE USUÁRIOS NO SISTEMA, BEM COMO OPERAÇÃO DE 
PESQUISA COM FILTROS, PAGINAÇÃO E ORDENAÇÃO DOS DADOS.

OS ENDPOINTS VOLTATOS PARA A ADMINISTRAÇÃO ESTÃO PROTEGIDOS 
CONTRA USUÁRIOS NÃO AUTENTICADOS

NO REQUISITO FUNCIONAL FOI SOLICITADO QUE AS AÇÕES DE 
CRIAÇÃO, EDIÇÃO E REMOÇÃO FOSSEM PROTEGIDOS PARA QUE SOMENTE
USUÁRIOS COM PERFIL ADMIN CONSEGUISSE ACESSAR. TODAVIA, MANTIVE
A PROTEÇÃO TAMBÉM AOS ENDPOINTS DE LISTAGEM E PESQUISA DE USUÁRIOS
POR CONSIDERAR QUE SERIA UMA FALHA DE SEGURANÇA TORNAR PUBLICO
INFORMAÇÕES DE USUÁRIOS DO SISTEMA.

ASSIM SÓ FICOU PÚBLICO O ENDPOINT DE LOGIN

ORA TODO O CÓDIGO FONTE ESTÁ SENDO DISPONIBILIZADO NO GITHUB.

FOI CRIADO O ARQUIVO stack.yml COM AS CONFIGURAÇÕES DA IMAGEM DOCKER
UTILIZADA PARA RODAR O MONGODB

PARA RODÁ-LA BASTA EXECUTAR O COMANDO:

docker compose -f stack.yml up

PARA QUE POSSA RODAR O SISTEMA E REALIZAR A AUTENTICAÇÃO E 
CONSUMIR OS ENDPOINTS, FAZ-SE NECESSÁRIO ANTES CADASTRAR UM 
USUÁRIO DIRETAMENTE NO BANCO

{"nome":"admin", "senha":"$2a$12$gIUpxvfn2kLYoCQ/rMycGeXJsOhTkKnL.03yr.Joc4W1MSzSezczm"}

ESSA SENHA FOI GERADA POR BCRYPT(admin) no site abaixo:
https://bcrypt-generator.com/

APOS ESSE CADASTRO JÁ TORNA-SE POSSÍVEL A AUTENTICAÇÃO COM ESTE 
MESMO USUÁRIO E UTILIZAÇÃO DOS ENDPOINTS

OS ENDPOINTS SÃO:

# PÚBLICOS
  /login
    - POST

# PROTEGIDOS
  /admin/user
    - GET
    - PUT
    - POST
    
  /admin/user/{IDUSER}  
    - DELETE
    
  /admin/user/buscar
    - POST

A APLICAÇÃO POSSUI DUAS FORMAS DE SER GERADA UMA IMAGEM DOCKER DELA
ATRAVÉS DO DOCKERFILE COM O COMANDO

docker build -f <NOME DA IMAGEM> .

E ATRAVÉS DO SPRING MAVEN PLUGIN QUE JÁ GERA UMA IMAGEM DOCKER
COM O COMANDO:

mvn spring-boot:build-image

TODOS ESSES COMANDO SENDO EFETUADOS NO TERMINAL NA RAIZ DO PROJETO

TAMBÉM COMO ATENDIMENTO DE REQUISITOS, FOI CRIADO O ARQUIVO
YAML QUE CONTÉM CONFIGURAÇÕES NECESSÁRIAS PARA O DEPLOYMENT 
NO KUBERNETS

TODAVIA NÃO FOI POSSIVEL COBRIR TODO O SISTEMA COM TESTES UNITÁRIOS,
EM VIRTUDE DE GRANDE PARTE DO TEMPO DO PRAZO TER SIDO GASTO TENTANDO
IMPLEMENTAR A SEGURANÇA DO SISTEMA ATRAVÉS DO SPRING SECURITY, UTILIZANDO
TOKEN JWT CRIPTOGRAFADO (JWE).

ASSIM DESDE JÁ AGRADEÇO A OPORTUNIDADE. JÁ FOI UM GRANDE APRENDIZADO,
ESPERO APRENDER AINDA MAIS, COMO O QUE VEM POR AÍ.
